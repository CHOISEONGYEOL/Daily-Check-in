## 1. 프로젝트 개요

| 항목 | 내용 |
|------|------|
| **목적** | 교실 출석 확인 겸 협동 미니게임 |
| **대상** | 학생 + 교사 (교사 계정: 학번 `77777`) |
| **플랫폼** | 웹 브라우저 (데스크톱 + 모바일) |
| **렌더링** | Canvas 2D |
| **네트워크** | Supabase Realtime Broadcast |
| **FPS** | 61 FPS (프레임 타임 스로틀링) |
| **게임 시간** | 5분 (300초) |

## 2. 시스템 아키텍처

### 모듈 믹스인 패턴

```
Game (core)
 ├── GamePhysics      → 중력, 충돌, 플랫폼 처리
 ├── GameMechanics    → 열쇠, 문, 발판, 엘리베이터, 블록
 ├── GameRender       → 캔버스 렌더링 파이프라인
 ├── GameAI           → NPC 의사결정, 이동
 ├── GameStages       → 스테이지 정의 (레이아웃, 오브젝트)
 ├── MazeGame         → 미로 게임 전용 로직
 ├── EscapeRoom       → 방탈출 게임 전용 로직
 ├── CrosswordGame    → 크로스워드 게임 전용 로직
 └── OllaOllaGame     → 올라올라 게임 전용 로직
```

`main.js`에서 `Object.assign(Game, ...)`으로 모든 믹스인을 하나의 Game 객체에 통합.

### 핵심 상수

```
월드: 1600×600 (기본) ~ 2400×900 (방탈출)
뷰포트: 800×600
중력: 0.5 | 점프: -10 | 이동속도: 3.2
카메라 줌: 1.6배
```

## 3. 게임 플로우

```
프로필 생성 → 로비 (캐릭터/상점/인벤토리)
                ↓
         OTP 인증 → 대기실 입장
                ↓
     메이플스토리풍 대기실 (자유 이동 + 공놀이)
                ↓
         교사가 "게임 시작" → 투표 오버레이
                ↓
     20초 투표 (NPC도 참여) → 동률 시 교사 타이브레이크
                ↓
         5초 카운트다운 → 게임 진입
                ↓
     ┌─ 피코파크 (협동 퍼즐)
     ├─ 넘버매치 (체크포인트 레이스)
     ├─ 미로 탈출 (1인칭 미로)
     ├─ 방탈출 (단서 + 퀴즈)
     ├─ 크로스워드 (한글 십자말풀이)
     └─ 올라올라 (OX 퀴즈 계단)
                ↓
     출석 확인 오버레이 (1.8초) → 5분 게임 플레이
                ↓
     스테이지 클리어 or 시간 초과
                ↓
     승리 연출 (3초) → 보상 지급 → 로비 복귀
```

## 5. 대기실 시스템

### 월드

- **크기**: 2000×900px, 카메라 줌 1.8배
- **구조**: 메이플스토리풍 다층 플랫폼 (나무, 마법, 자연)
- **관람석**: 3개 박스 (엘리베이터로 접근, 벽으로 격리)
- **엘리베이터**: 6개 (상행 3 + 하행 3)

### 기믹 (랜덤 장애물)

| 기믹 | 효과 |
|------|------|
| 바람 | 이동속도 50% 감소 |
| 화면 흔들림 | 시각 효과 |
| 중력 반전 | 위아래 뒤집힘 |
| 조작 반전 | 좌우 키 반대 |
| 화면 뒤집기 | 좌우 시각 반전 |
| 정전 | 원형 암흑 비네팅 |
| 크기 변환 | 플레이어 확대/축소 |
| 유령 플랫폼 | 투명 트램폴린 |

### 팀 공놀이

- 2팀 자동 편성 (좌/우)
- 공 물리: 탄성 0.5, 마찰 0.985, 중력 0.7
- 4개 골존 (바닥 2 + 관람석 천장 2)
- 먼저 다수 골 넣는 팀 승리

### 소셜 기능

- **채팅**: 30자 제한, 말풍선 표시 (180프레임 유지)
- **이모트**: 😐 무표정 / 😮 팽창 / 💥 폭발 (키 1/2/3)

## 7. 게임 모드 상세

### 7-1. 🔑 피코파크 (Pico Park)

> 25인 협동 플랫포머. 함께 열쇠를 모아 문을 열고 탈출.

**3 스테이지 구성:**

| 스테이지 | 제목 | 크기 | 열쇠 | 핵심 메커니즘 |
|----------|------|------|------|---------------|
| 1 | 열쇠를 모아라! | 1600×600 | 2 | 발판 → 다리, 엘리베이터 |
| 2 | 분업과 협동! | 2000×600 | 4 | 블록 밀기, 이중 발판, 스태킹 우물 |
| 3 | 최종 관문! | 2400×700 | 4 | 3연속 발판, 블록+엘리베이터, 정상 등반 |

**핵심 메커니즘:**

| 오브젝트 | 작동 방식 |
|----------|-----------|
| **발판(Plate)** | N명 이상 올라서면 연결된 다리 출현 |
| **다리(Bridge)** | 발판 활성화 시 나타나는 플랫폼 |
| **엘리베이터** | N명 탑승 시 상승, 미달 시 하강 |
| **블록 밀기** | N명이 같이 밀면 이동 (6인 기준) |
| **스태킹** | 플레이어가 서로 머리 위에 올라서서 높은 곳 도달 |
| **열쇠** | 게이트 조건 충족 시 수집 가능, 모두 모으면 문 열림 |
| **위험요소** | 가시/용암 - 즉사 후 60프레임 뒤 리스폰 |

**인원 스케일링:**

| 인원 | 열쇠 수 | 협동 인원 계산 |
|------|---------|----------------|
| 1명 | 1 | 모든 게이트 해제 |
| 2-3명 | 1 | 모든 게이트 해제 |
| 4-5명 | 2 | `round(base × n/25)` |
| 6-10명 | 3 | 〃 |
| 11-15명 | 4 | 〃 |
| 16-20명 | 5 | 〃 |
| 21+명 | 6 | 〃 |

### 7-2. 🔢 넘버매치 (오리엔티어링)

> 각자 배정받은 체크포인트를 순서대로 방문하는 레이스.

**3 스테이지 구성:**

| 스테이지 | 특징 | 체크포인트 |
|----------|------|-----------|
| 1 | 기초 점프 코스 | 25개 중 3~5개 배정 |
| 2 | 수직 + 수평 혼합 | 25개 |
| 3 | 미로형 지형, 4층 구조 | 25개 (최고 난이도) |

- 체크포인트 Fisher-Yates 셔플로 랜덤 배정
- 현재 타겟: 금색 표시, 완료: 체크마크
- 전원 완료 시 문 개방

### 7-3. 🌀 미로 탈출 (Maze)

> 1인칭 시점 미로 탐험. 출구를 찾아 탈출.

**적응형 난이도:**

| 인원 | 격자 크기 | 시야 반경 | 플래시 시간 |
|------|-----------|-----------|-------------|
| 1-5명 | 11×8 | 160px | 240프레임 |
| 6-10명 | 15×11 | 130px | 210프레임 |
| 11-15명 | 19×14 | 110px | 195프레임 |
| 16-20명 | 22×16 | 95px | 180프레임 |
| 21+명 | 25×18 | 80px | 180프레임 |

**게임 흐름:**
1. **미로 공개 (플래시)**: 전체 미로를 잠깐 보여줌 → "미로를 기억하세요!"
2. **탐험**: 제한된 시야 반경 내에서 이동
3. **승리**: 첫 번째 플레이어가 출구 도달 시 전원 클리어

**미로 생성**: Recursive Backtracker 알고리즘 (완전 미로 - 해 1개)

### 7-4. 🚪 방탈출 (Escape Room)

> 포켓몬스터 스타일 맵 탐험 + 단서 수집 + 발판 퀴즈.

**2개 방 구성:**

| 방 | 제목 | 크기 | 특징 |
|----|------|------|------|
| 1 | 비밀의 방 | 2000×800 | 3단 플랫폼, 5개 퀴즈 슬롯 |
| 2 | 최종 탈출 | 2400×900 | 위험요소 추가 (가시/용암) |

**퀴즈 시스템:**
- 6가지 테마: 색깔, 숫자, 도형, 동물, 카드, 계절
- 각 퀴즈: 4개 발판 중 **정답 3개를 순서대로 밟기**
- 단서를 먼저 찾아야 정답 순서를 알 수 있음
- 오답 시 전체 진행 초기화 + 에러 타이머

**핵심 메커니즘:**
- **단서**: 맵 곳곳에 숨김, 접근 시 자동 발견 (펄스 애니메이션)
- **유령 모드**: 잔여 60초에 활성화 → 죽은 플레이어도 발판 밟기 가능
- **문 잠금**: 모든 퀴즈 해결 전에는 문이 플레이어를 밀어냄

---

### 7-5. 📝 크로스워드 (Crossword)

> 한글 십자말풀이 퀴즈를 시간 내에 풀기.

**5 스테이지 구성:**

| 스테이지 | 주제 | 격자 | 단어 수 |
|----------|------|------|---------|
| 1 | 학교생활 | 5×5 | 6 |
| 2 | 자연과 생활 | 7×7 | 7 |
| 3 | 음식과 요리 | 7×7 | 7 |
| 4 | 지리와 세계 | 8×8 | 8 |
| 5 | 문화와 예술 | 8×8 | 9 |

> 인원별 조정: ≤5명 → 3스테이지, ≤15명 → 4스테이지, >15명 → 5스테이지

**입력 방식:**
- 셀 클릭 → 방향(가로/세로) 자동 선택 + 토글
- 데스크톱: 키보드 입력
- 모바일: 한글 IME 키보드 오버레이
- 교차점 잠금: 이미 맞은 교차점 글자는 고정

**멀티플레이어:**
- 리더보드 (우하단): 풀이 수 기준 정렬
- 0.5초 간격 진행도 브로드캐스트

---

### 7-6. 🧗 올라올라 (Olla Olla)

> OX 퀴즈를 맞혀 10단 계단을 올라가는 서바이벌.

**게임 설정:**

| 항목 | 값 |
|------|-----|
| 총 계단 | 10단 |
| 문제당 제한시간 | 10초 |
| 초기 목숨 | 3개 ❤️ |
| 게임 제한시간 | 5분 |

**퀴즈 뱅크 (총 65문제):**

| 분류 | 문제 수 | 예시 |
|------|---------|------|
| 일반상식 | 10 | "대한민국 수도는 서울이다" |
| 역사 | 10 | "임진왜란은 1592년에 일어났다" |
| 과학 | 10 | "물의 화학식은 H2O이다" |
| 수학 | 8 | "삼각형의 내각의 합은 180도이다" |
| 지리 | 8 | "세계에서 가장 큰 대양은 태평양이다" |
| 문화/생활 | 8 | "올림픽은 4년마다 열린다" |

**게임 플로우:**
1. **Ready** (1.5초): 준비 카운트다운
2. **Question** (10초): 문제 출제 + 타이머 바
3. **Answer**: O/X 선택 (키보드: O/X/1/2, 마우스: 버튼)
4. **Result** (1.5초): 정답 공개 + 올라가기/떨어지기 애니메이션
5. **Clear**: 10단 도달 시 승리
6. **Fail**: 목숨 0 시 탈락

**스코어링:**
- 기본: 100점/정답
- 콤보 보너스: +20점 × 콤보 스택
- 최고 콤보 추적

## 8. AI / NPC 시스템

### NPC 생성

- **솔로 모드**: 5~24명 AI NPC 자동 생성
- **멀티플레이어**: NPC 대신 실제 원격 플레이어 (AI 스킵)

### 모드별 AI 전략

#### 피코파크: 가중치 기반 작업 분배

```
_buildPicoJobs()   → 현재 상황 분석, 작업 풀 생성
_assignPicoJob()   → 가중치 비례 NPC 배정
_moveNPC()         → 배정된 작업에 따른 이동
```

| 작업 | 가중치 | 행동 |
|------|--------|------|
| 스태킹 (열쇠) | 8 | 열쇠 아래 집결, 60% 토대/40% 등반 |
| 발판 유지 | 3 | 발판 위에 서서 대기 |
| 엘리베이터 탑승 | 동적 | 필요 인원만큼 배정 |
| 블록 밀기 | 동적 | 블록 쪽으로 이동 후 밀기 |
| 열쇠 대기 | 1 | 게이트 해제 대기 |
| 문으로 이동 | - | 모든 열쇠 수집 후 |

#### 넘버매치: 체크포인트 순차 추적
- 배정된 체크포인트로 순서대로 이동
- 지능 변수에 따라 혼란도 차등화

#### 방탈출: 단서 → 퀴즈 → 문
- 60% 확률로 미발견 단서 탐색
- 발판 순서대로 밟기
- 지능에 따라 방황 빈도 변동

#### 미로: 랜덤 방향 전환
- 60~180프레임마다 방향 변경
- 30% 확률 정지
- 속도: 0.6~1.4 (개인차)

#### 올라올라: 정답률 기반
- AI 정답률: 50~90% (개인별)
- 최대 15명 NPC 참여

### NPC 채팅

상황별 대사 풀 (한국어):
- 열쇠: "열쇠 어딨지?" / "위에 있나?"
- 스태킹: "올라타!" / "내 위로 와!"
- 문: "문으로 가자!" / "빨리!"
- 발판: "여기 밟아!" / "누르고 있어!"
- 블록: "같이 밀자!" / "더 힘!"

## 9. 물리 엔진

### 기본 물리

```javascript
중력:     vy += 0.5 (매 프레임)
최대 낙하: vy 상한 14
이동:     x += vx, y += vy
월드 경계: x = [w/2, W-w/2], y > H+50 시 즉사
```

### 플랫폼 충돌

| 유형 | 처리 |
|------|------|
| **바닥/벽** | AABB 수평 밀어내기 |
| **일반 플랫폼** | 상단 착지만 (vy≥0 + 수평 범위 내) |
| **다리** | 플랫폼과 동일, `visible=true`일 때만 |
| **엘리베이터** | 다리와 동일 + 수직 이동 제공 |

### 엔티티 간 충돌 (스태킹)

- AABB 오버랩 검사
- 수직 오버랩 < 수평 → 머리 위 착지 (스태킹)
- 수평 오버랩 < 수직 → 좌우 밀어내기 (마찰 0.3)
- 문 진입/사망 엔티티 제외

### 원격 플레이어

- 로컬 물리 **완전 스킵** (`if(e.isRemote) return`)
- 네트워크에서 받은 위치 직접 적용 (예측 없음)

## 12. 교사 시스템

### 갓 모드 (관전)

- 캐릭터 없이 자유 카메라 이동
- 줌: 0.4x ~ 3x (±키)
- **자유 카메라** (ESC): WASD 이동
- **POV 모드** (F 토글, TAB 순환): 학생 시점 추적
- 화면 클릭으로 학생 선택

### 게임 시작 흐름

```
교사 "게임 시작" 클릭
  → DB 시그널 전체 학생 전송
  → 학생들 투표 오버레이
  → 교사 실시간 투표 현황 확인 (읽기 전용)
  → 동률 시 교사가 타이브레이크 선택
  → DB로 선택 게임 브로드캐스트
  → 5초 카운트다운 → 게임 시작
```

### 출석 관리

- 대기실 입장 시 자동 체크인
- 교사 대시보드에서 학생 목록 확인
- 참여자 수 DB 연동 (미연결 시 수동 입력)
